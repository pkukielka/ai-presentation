What AI Can Do For You

And Where It Falls Short

Code Quality & Testing

AI Assistance for Code Quality

  • Adding more tests - both for existing and new code
  • Adding Documentation & Comments
  • Suggesting Refactoring & Improvements
  • Identifying Potential Bugs & Edge Cases

Unit Testing Limitations

  • Often misses critical corner/edge cases
  • Agentic AI can "fix" tests to pass (instead of fixing code)
  • Generates overly specific or brittle tests
  • Tests might rely on AI's own incorrect code assumptions
  • Requires human expertise to validate test coverage

Debugging & Error Analysis

AI Assistance for Debugging

  • Explaining Cryptic Error Messages
  • Suggesting Debugging Steps
  • Understanding Unfamiliar Code/Traces

Limitations in AI Debugging

  • Cannot perform live debugging (breakpoints, runtime state)
  • Relies on user-provided, potentially incomplete context
  • Suggestions can be generic or mislead
  • Struggles with environment/runtime-specific issues
  • Limited integrated debugging tools beyond chat

Code Generation & Workflow Automation

Boosting Your Workflow Efficiency

  • Generating Boilerplate Code
  • Creating Complex Shell Commands/Scripts
  • Providing API Usage Examples
  • Assisting with Configuration Files
  • Doing code reviews
  • Leading semi-automatic migrations and rewrites

Code Correctness & Hallucination Risks

  • Generates plausible-but-wrong code
  • Hallucinates APIs or library behavior
  • Introduces subtle logic errors
  • Errors often hide in edge cases
  • Requires rigorous verification & debugging

Understanding & Exploration

AI for Learning & Exploration

  • Summarizing code functionality
  • Making bigger projects easier to understand
  • Getting started with new projects or technologies
  • Exploring alternative implementations
  • Real-time learning during implementation

Context & Attention Challenges

  • Limited effective context window
  • Attention dilution with long context (important info gets lost)
  • Agentic context fetching helps, but still complex
  • Difficulty managing state & history across turns
  • Requires manual intervention or careful prompting

Communication & Explanation

AI Assistance for Communication

  • Drafting commit messages
  • Drafting PRs summaries
  • Explain technical concepts
  • Assist with code review feedback (drafting)
  • Help with organisation and summarisation of knowledge

Lack of "Why" & Deep Reasoning

  • Often provides code without explaining underlying logic
  • Struggles to justify design choices or alternatives
  • Less effective for learning *how* to think about problems
  • Can generate working but architecturally poor code

Cognition Enhancement vs. Over-Reliance

Augmenting Your Cognitive Flow

  • Amplifying your existing knowledge
  • Exploring unfamiliar technologies
  • Focusing on higher-level concerns

Risks of Over-Reliance & Skill Degradation

  • Can hinder deep understanding of fundamentals
  • Risk of blindly accepting incorrect suggestions
  • Reduced ability to debug complex issues independently
  • Becoming an "AI operator" vs. a problem solver

Security Assistance & Risks

AI Assistance for Security

  • Identify potential common vulnerabilities (initial pass)
  • Explain security risks of code patterns
  • Suggest secure coding practices/alternatives
  • Help understand security concepts

AI-Introduced Security Risks

  • Can generate code with common flaws (injection, XSS, etc.)
  • Risk of exposing proprietary code via cloud APIs
  • Training data leakage concerns (depending on vendor policy)
  • Requires dedicated human security review, not just AI check

Tips & Tricks

Work Planning and Verification

  • Ask AI to summarise its understanding of the problem
  • Ask additional questions for clarification
  • Ask AI to suggest potential solutions, with their benefits and drawbacks
  • Make sure answers describe both 'why' and 'how'
  • Choose best solution and ask AI to outline the implementation

Task Decomposition

  • Break complex problems into AI-solvable units
  • Create clear specifications for each component
  • Manage dependencies between tasks
  • Maintain the big picture

Implementation verification

  • DO NOT accept solution you do not understand
  • Ask AI to write tests for the implementation
  • Carefully verify that they cover all corner cases
  • Make sure some integration tests are included - AI tends to focus on easier unittests
  • Spend saved time on more end-to-end testing

Practical tips

  • Try to use AI very early in the process even if you feel it won't be able to handle it
  • Use AI for tasks which are hard to figure out or implement, but easy to verify, not vice versa
  • Use 'git add .' to save milestones during more complex AI edits
  • Use scratch pads or dedicated branches for experimentation
  • Use context accumulated during AI session to generate great commit messages, PR descriptions and QA testing scripts
