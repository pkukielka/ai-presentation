Fixing Configuration Settings in JetBrains Plugin

Real conversation with Claude for implementing a migration:

  • Analyze existing codebase and problem
  • Design solution for fixing configuration issues
  • Implement the migration class
  • Test and refine implementation
  • Generate a proper PR description

Conversation Example

That is correct! This fix is almost done, but some harm was already done and incorrect configurations were written on the client machines. I want to clean them up when new version of our plugin will be run for the first time.

I'll add a migration to clean up incorrect configurations from the cody_settings.json file. Let me first check the existing migration code and the affected settings file.

After reviewing the code and settings, I'll create a new migration class to clean up these settings.

Real PR Outcome

PR #7904 on sourcegraph/cody


class ClientConfigCleanupMigration : Migration {
    override fun run(project: Project): Boolean {
        val configService = CodyConfigService.getInstance(project)
        val config = configService.config
        
        // Remove temporary client-side values
        var modified = removeIfExists(config, "cody.advanced.agent")
        modified = removeIfExists(config, "cody.advanced.hasNativeWebview") || modified
        
        // Remove default values to clean up config file
        modified = cleanupDefaultValues(config) || modified
        
        if (modified) {
            configService.saveConfig()
        }
        
        return modified
    }
}
